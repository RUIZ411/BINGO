# SUWEET BINGO - Room Version

## 추가된 방 구조

- 홈페이지에서 4개 방을 선택합니다.
  - 1번 방
  - 2번 방
  - 3번 방
  - 수힛방
- 방 생성 시 관리자 코드가 필요합니다.
- 방 생성 후 5자리 랜덤 입장 코드가 발급됩니다.
- 이후 접속자는 해당 5자리 코드로 방에 입장해 빙고판을 조작할 수 있습니다.
- 각 방의 빙고판, 체크 상태, 치킨 수, 현상금, 메모장은 서로 독립 저장됩니다.

## 관리자 코드 관리

관리자 코드는 `app.js` 상단의 `MASTER_ROOM_ADMIN_CODE` 값으로 관리합니다. 공개 화면이나 README에는 실제 코드를 적지 않는 것을 권장합니다.

## 화면 주소

홈페이지:

```txt
index.html
```

방 접속 후에는 자동으로 아래 구조를 사용합니다.

```txt
index.html?room=room1&view=public
index.html?room=room1&view=obs
```

## Firebase Rules 중요

방 코드 방식은 GitHub Pages 정적 사이트에서 동작하는 간편 권한 구조입니다. 방 코드로 접속한 사용자가 Firebase에 저장할 수 있어야 하므로 Realtime Database Rules는 아래처럼 설정해야 합니다.

```json
{
  "rules": {
    "bingoRooms": {
      ".read": true,
      ".write": true
    }
  }
}
```

이 방식은 방송/지인 운영용 간편 코드 방식입니다. 방 코드가 공유되면 같은 방을 조작할 수 있습니다.

## 포함 기능

- 숫자 / 미션 / 알파벳 / 리셋 빙고
- 숫자 빙고 5×5 / 7×7 / 10×10
- 미션 / 알파벳 / 리셋 5×5 고정
- 현상금 대상 칸 + 현상금 개수 표시
- 칸 수정 모드
- 메모장 팝업
- 리셋 빙고 메뉴판
- 리셋빙고 전용 줄/전체 체크 해제
- 빙고 타입별 관리자 메뉴 자동 정리
- 송출용 화면 + 비율 조절
- 방별 독립 저장

## 홈페이지 사용 방법 안내 추가

방 목록 아래에 사용 방법 섹션을 추가했습니다.

- 방 생성: 별도로 안내받은 관리자 코드 입력
- 입장 코드: 생성된 5자리 코드 공유
- 빙고판 조작: 입장 후 기존 기능 사용
- 송출용: 각 방의 송출용 버튼으로 OBS 화면 진입


## 관리자 아이디/비밀번호 방 생성 인증

이번 버전부터 방 생성/초기화/코드 재발급은 화면에 노출되는 관리자 코드가 아니라 관리자 아이디와 비밀번호로 인증합니다.

- 아이디 입력칸에는 `suweet`처럼 앞부분만 입력할 수 있습니다.
- 코드 내부에서 자동으로 `@suweet.com`을 붙여 `suweet@suweet.com` 형식으로 인증합니다.
- 전체 이메일을 직접 입력해도 그대로 사용할 수 있습니다.
- 현재 구현은 Firebase Auth 이메일/비밀번호 인증을 사용합니다. Firebase Functions 검증 함수를 따로 연결하려면 `app.js`의 `verifyRoomAdminCredentials(email, password)` 함수 안만 교체하면 됩니다.

## 이번 수정 내용

- 숫자 랜덤 추첨 메뉴와 관련 코드를 제거했습니다.
- 관리자 패널 메뉴는 현재 빙고 타입에 맞는 항목만 보이도록 정리했습니다.
  - 미션: 방 권한 → 빙고 생성 → 칸 수정 모드 → 현상금 설정 → 치킨 수 → 미션 일괄 입력 → 전체 관리
  - 숫자/알파벳: 방 권한 → 빙고 생성 → 칸 수정 모드 → 현상금 설정 → 치킨 수 → 전체 관리
  - 리셋: 방 권한 → 빙고 생성 → 칸 수정 모드 → 현상금 설정 → 치킨 수 → 리셋 빙고 메뉴판 → 전체 관리
- 리셋빙고 메뉴판 안에 가로 5줄, 세로 5줄, 대각선 2줄, 전체 체크 해제 버튼을 추가했습니다.
- 체크 해제 버튼은 체크를 추가하지 않고 이미 체크된 칸만 체크 해제합니다.
