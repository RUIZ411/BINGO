# SUWEET BINGO V1 - SOOP 대결미션 1차 테스트 버전

이 버전은 실제 SOOP/SSAPI 서버 연결 전에, 방별 API 설정과 체크 해제 동작을 사이트 안에서 먼저 검증하는 1차 버전입니다.

## 포함 기능

- 방 4개 구조: 1번 방 / 2번 방 / 3번 방 / 수힛방
- 방 생성 관리자 인증: 아이디 입력 시 `@suweet.com` 자동 추가
- 5자리 방 입장 코드
- 빙고 생성/체크/복구/수정 모드
- 현상금 기능
- 메모장
- 송출용 화면
- 리셋 빙고 메뉴판: 한개 / 한줄 / 전체
- SOOP 대결미션 1차 테스트 패널

## SOOP 대결미션 1차 테스트 방식

관리자 패널의 `SOOP 대결미션 테스트`에서 다음을 설정합니다.

1. `연동 사용` 체크
2. 스트리머 아이디 입력
   - 여러 명이면 한 줄에 하나씩 입력하거나 쉼표로 구분
3. `연동 설정 저장`
4. 테스트 스트리머 아이디와 펀딩 개수 입력
5. `테스트 실행`

동작 규칙은 다음과 같습니다.

- 테스트 펀딩 개수 === 한개 값 → 체크되어 있는 칸 중 랜덤 1칸 체크 해제
- 테스트 펀딩 개수 === 한줄 값 → 체크되어 있는 칸이 포함된 랜덤 한 줄 체크 해제
- 테스트 펀딩 개수 === 전체 값 → 전체 칸 체크 해제
- 그 외 값 → 아무 동작 없음

누적 계산이나 배수 계산은 하지 않습니다. 정확히 일치할 때만 실행됩니다.

## Firebase Rules

현재 방 코드 방식과 1차 테스트 기능을 사용하려면 Realtime Database Rules는 아래처럼 둡니다.

```json
{
  "rules": {
    "bingoRooms": {
      ".read": true,
      ".write": true
    }
  }
}
```
